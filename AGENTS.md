# LocalVault Coding Rules

- If a coding task is unclear, ask concise clarifying questions before implementing it.
- Avoid assumptions. When an assumption is necessary to make progress, state it explicitly in the response before relying on it.
- Prefer the smallest complete implementation. If the work can be done in about 100 lines of code, do not produce a 200-line solution; avoid unnecessary abstractions, features, and duplication.
